# 🤝 AltruBot: Система координации волонтеров (MVP)

Данный проект представляет собой концепт  Telegram-бота, созданного для приема заявок от граждан, нуждающихся в помощи, и их координации администраторами/волонтерами. 

Бот написан с использованием современных стандартов разработки под Telegram: асинхронность, модульная архитектура и использование машины состояний (FSM) для пошагового сбора данных.

## 🌟 Основной функционал

**Для пользователей:**
- 📝 Пошаговое создание заявки на помощь (Машина состояний).
- 📍 Выбор категории проблемы и района города (через Reply-клавиатуры).
- 📸 Возможность прикрепить фотографию проблемы.
- 💬 Оставление контактных данных для связи.

**Для администраторов:**
- 🔒 Защищенный доступ (проверка по Telegram ID).
- 📥 Просмотр списка новых (необработанных) заявок.
- 🖼 Просмотр прикрепленных фотографий прямо в чате.
- ✅ Закрытие заявок в один клик (Inline-кнопки) с автоматическим обновлением базы данных.

## 🛠 Технологический стек
- **Язык:** Python 3.10+
- **Фреймворк:** [aiogram 3.x](https://docs.aiogram.dev/) (Асинхронное взаимодействие с Telegram API)
- **База данных:** SQLite + `aiosqlite` (Асинхронные SQL-запросы)
- **Конфигурация:** `python-dotenv` (Безопасное хранение токенов)

## 📁 Структура проекта
Проект разделен на логические модули для удобства поддержки и расширяемости:

```text
├── database/
│   └── db.py               # SQL-запросы (CRUD операции) и инициализация БД
├── handlers/
│   ├── user_request.py     # FSM: Логика пошагового сбора заявки от пользователя
│   └── admin.py            # Логика панели администратора
├── keyboards/
│   ├── reply.py            # Обычные текстовые кнопки (районы, категории)
│   └── inline.py           # Кнопки под сообщениями (закрытие заявки)
├── photos/                 # Директория для сохранения присланных фотографий
├── .env                    # Секретные ключи (токен, ID админа) - ИГНОРИРУЕТСЯ В GIT
├── config.py               # Глобальные настройки проекта (списки районов и т.д.)
├── requirements.txt        # Зависимости проекта
└── main.py                 # Точка входа: запуск бота и подключение роутеров
```

Плюсы текущей реализации
Модульность: Разделение логики (handlers, keyboards, database) позволяет легко добавлять новые функции, не ломая старый код.
Асинхронность: Бот не блокируется при чтении из БД или скачивании фото, что позволяет обслуживать сотни пользователей одновременно.
Безопасность SQL: Использование параметризованных запросов (?) в aiosqlite полностью исключает риск SQL-инъекций.
Масштабируемость интерфейса: Районы и категории вынесены в config.py. Чтобы добавить новый район, не нужно переписывать логику кнопок или БД

Текущие ограничения
Локальная БД (SQLite): Идеально для прототипа, но при масштабировании на несколько серверов потребуется переход на внешнюю СУБД.
Хранение файлов локально: Сохранение фото в папку photos/ делает невозможным деплой на эфемерные облака (Heroku, Render). При перезапуске сервера файлы теряются.
Отсутствие веб-панели: Аналитика и сложная статистика пока недоступны
